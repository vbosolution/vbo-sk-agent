      ▶ Best Practices
    

    

      

        
1. Quick POC via command.rb

        
For simple ideas, skip creating a full plugin — just prototype directly through the bridge:

        

          Prompt
          "I want a tool that selects all faces with area > 10m² and colors them red. Can you prototype this directly via command.rb first?"
        

        
If it works, then ask AI to turn it into a proper plugin. If not, you saved time discovering it early.

      

      

        
2. Develop your plugin normally

        
Create your plugin files in the Plugins folder. Edit code with your AI assistant as usual.
           The bridge is not where you write your plugin — it's for running and testing it.

      

      

        
2. Reload after changes

        
After editing plugin files, ask AI to reload them via the bridge:

        

          Prompt
          "Reload my_plugin/main.rb into SketchUp and show me any errors"
        

      

      

        
3. Debug with confirmation

        
For testing code that modifies the model, ask AI to show you the code first:

        

          Prompt
          "Test the draw_pipe method — show me the test code before running it"
        

      

      

        
4. Auto-fix loop (with limits)

        
Let AI fix bugs, but always set a retry limit:

        

          Prompt
          "Run this code in SketchUp. If it fails, read the error, fix it, and try again. Maximum 3 attempts — if still failing, stop and explain what's wrong instead of retrying."
        

        
Always set a limit. Never let AI retry endlessly — it may corrupt your model.

      

      

        
5. Inspect the model

        

          Prompt
          "How many groups are in my model? What materials are used?"
        

      

    

  


  
  

    

      ▶ Beyond Coding — Your AI Assistant
    

    

      
SkAgent isn't just for plugin development. AI becomes a powerful assistant that can act on your model directly:

      

        
Generate reports

        
Prompt
          "Create an HTML report of all selected groups/components with columns: name, bounding box dimensions, tag, and material. Display it in a SketchUp dialog (UI::HtmlDialog)."

      

      

        
Draw from reference images

        
Prompt
          "Read this land survey image [attach file] and draw the plot shape in SketchUp. Move it near the origin."

      

      

        
Batch-edit scenes

        
Prompt
          "Copy the camera angle from the current scene and apply it to all scenes whose name contains 'Plan'."

      

      

        
Translate entire model

        
Prompt
          "Translate all text entities in this model (3D text, screen text, dimensions, labels) into Japanese."

        
Hours of manual work for a human — seconds for AI with the bridge.

      

      

        
Model cleanup

        
Prompt
          "Find all ungrouped edges and faces in the model, group them by proximity, and name each group."

      

      
Anything you can do in SketchUp's Ruby Console, your AI can now do for you — but at a higher level. No code needed, just describe what you want in plain language.
